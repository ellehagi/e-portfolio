Inital Post:In table 1 of Padhy et al. (2021), the article states that various factors influence the reusability of object-oriented software components. The first component is modularity, in highly modular code, the code is usually easier to understand, test and use in other systems, this allows for components to be used independently. Cohesion in code increases reusability in the code as the classes tend to have a clear and focused purpose. Moreover, Low coupling is generally very desirable as the components donâ€™t rely heavily on one and another and Lower complexity is regarded as good because the components are easier to adapt and understand. Well-documented code increases its reusability because it allows developers to understand the code. Further, smaller components tend to be more reusable because they are not tightly bound to very specific use cases. The proper usage of inheritance can promote reusability.
